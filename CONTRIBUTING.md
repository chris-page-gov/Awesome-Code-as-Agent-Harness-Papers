# Contributing

Thank you for your interest in contributing to **Awesome Code as Agent Harness Papers**! This curated list grows with the community, and we welcome additions, corrections, and reorganizations.

## How to Contribute

### Adding a paper

1. Identify which subsection your paper belongs to (Harness Interface / Mechanisms / Scaling / Applications). If unsure, open an issue first to discuss.
2. Open a pull request that adds a row to the appropriate Markdown table:

   ```markdown
   | [Paper Title](https://arxiv.org/abs/XXXX.XXXXX) | Venue Year |
   ```

3. Use the **canonical link** for the paper:
   - Prefer the arXiv abstract page (`https://arxiv.org/abs/XXXX.XXXXX`)
   - For published-only papers, use the official venue URL or DOI (`https://doi.org/...`)
   - For project-style entries (blogs, model cards, demos), use the canonical project URL
4. Keep titles short and faithful to the paper's title (no marketing words).
5. Try to keep the rows in chronological order within each subsection where possible.

### Reporting an issue

Open a GitHub issue if you spot:

- A broken link
- A wrong categorization
- A missing significant paper
- A typo or formatting bug

Please include enough context so a maintainer can act without extensive triage.

### Style guide

- One paper per row, one venue per row.
- Use US English.
- Avoid emojis inside the table rows.
- Keep the table header consistent: `| Paper | Venue/Year |`.

### Maintainers

This list is maintained by the authors of the *Code as Agent Harness* survey. See the **Citation** section in the [README](README.md) for details.
